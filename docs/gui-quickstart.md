# 桌面版和网页版扩展实现指南

## 🎯 扩展策略总览

我已经为你设计了完整的GUI扩展方案，核心思想是：**复用现有核心业务逻辑，只添加不同的UI层**。

## 📋 已实现的原型

### 1. 桌面版GUI原型 (Tkinter)
**位置**: `llm_translate/gui/main.py`

**功能特性**:
- ✅ 项目管理（创建、查看、选择）
- ✅ 翻译执行（后台异步处理）
- ✅ 进度监控（进度条、状态显示）
- ✅ 日志查看（实时日志、保存功能）
- ✅ 结果导出
- ✅ 标签页界面（项目管理、翻译任务、运行日志）

**启动命令**:
```bash
# 安装依赖（如果需要）
pip install llm-translate[gui]

# 启动桌面版
python -m llm_translate.gui.main
# 或者
llm-translate-gui
```

### 2. 网页版原型 (FastAPI)
**位置**: `llm_translate/web/app.py`

**API端点**:
- `GET /` - 主页
- `GET /api/projects` - 获取项目列表
- `POST /api/projects` - 创建新项目
- `POST /api/projects/{id}/translate` - 开始翻译
- `GET /api/projects/{id}/progress` - 查看进度
- `GET /api/projects/{id}/export` - 导出结果
- `GET /docs` - Swagger API文档

**启动命令**:
```bash
# 安装依赖
pip install llm-translate[web]

# 启动网页版
python -m llm_translate.web.app
# 或者
llm-translate-web

# 访问
# http://localhost:8000 - 主页
# http://localhost:8000/docs - API文档
```

## 🔌 核心集成点

### 与现有系统的集成
```python

from llm_translate.service import TranslationService
from llm_translate.config import Settings

# 在GUI中使用
settings = Settings.from_env()
service = TranslationService(settings)

# 所有现有功能都可用
project = service.create_project(...)
service.parse_project(project.id)
service.prepare_project(project.id)
service.translate_project(project.id, provider)
paths = service.export_project(project.id)
```
